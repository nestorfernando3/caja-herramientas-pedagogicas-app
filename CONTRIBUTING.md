# Contribuir al Proyecto

## Flujo tecnico
1. Crear rama desde `main` con prefijo `codex/`.
2. Implementar cambios pequenos y coherentes.
3. Ejecutar validaciones:
```bash
npm run lint
```
4. Crear commit con mensaje claro.
5. Abrir Pull Request.

## Criterios de aceptacion
- Sin errores de sintaxis.
- Sin regresiones en formularios ni panel editorial.
- Documentacion actualizada cuando cambie comportamiento.

## Convenciones
- Mantener lenguaje en espanol en UI y docs.
- Usar nombres descriptivos en variables y endpoints.
- Evitar dependencias innecesarias.
